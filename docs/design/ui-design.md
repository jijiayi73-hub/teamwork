# UI 设计文档

项目名称：心灵情绪日记 / Mindful Memory Diary / Inner Garden

## 1. Home

### 页面目标

首页用于让用户快速理解产品，并降低开始记录情绪的心理压力。用户进入后应立即知道：这里可以倾诉今天的心情，AI 会温柔陪伴并整理成日记，日记会保存为 Memory Garden 中的记忆卡片。

首页也需要明确产品边界：本产品不是心理诊断工具，只做情绪记录、陪伴和回忆整理。

### 页面布局

- 顶部导航栏：左侧为 Logo 和产品名“心灵情绪日记”，右侧为“Memory Garden”“关于产品”“开始记录”。
- 首屏主区域：展示主标题“把今天的情绪，安静地放进一座记忆花园。”。
- 副标题：说明用户可以上传图片或对 AI 说说今天发生了什么，AI 会像朋友一样陪伴整理。
- 边界说明：用轻量文案提示“这里不是心理诊断工具，只是陪你记录和整理今天的情绪。”。
- 主操作区：提供“开始今天的记录”和“进入 Memory Garden”。
- 下方说明区：三张小卡片展示“倾诉、陪伴、保存”三步流程。
- 底部安抚文案：“这里不是评判你的地方，只是陪你把今天慢慢放下来。”。

### 麦克风权限与录音状态

由于本页面支持语音输入，用户首次点击语音输入按钮时，系统需要请求麦克风权限。

交互流程：

- 用户点击语音输入按钮。
- 页面先弹出说明提示：“为了记录你的语音倾诉，我们需要使用麦克风权限。录音内容仅用于转写和生成日记，不会公开展示。”
- 用户点击“允许使用麦克风”后，浏览器弹出系统麦克风授权窗口。
- 用户同意后，进入录音状态。
- 用户拒绝后，页面提示：“无法使用麦克风，你仍然可以通过文字输入继续记录。”

录音状态：

- 未录音：显示麦克风按钮。
- 请求权限中：显示“正在请求麦克风权限……”。
- 录音中：麦克风按钮高亮，显示“正在聆听……”。
- 录音结束：将语音转为文字，并显示在输入框或聊天记录中。
- 权限被拒绝：隐藏录音状态，保留文字输入方式。

### 主要按钮

- 开始今天的记录：进入 AI Companion Chat。
- 进入 Memory Garden：进入记忆花园页。
- 语音输入：用户点击后先请求麦克风权限，授权成功后开始录音；如果用户拒绝授权，则提示用户可以改用文字输入。
- 查看示例日记：可选按钮，用于展示示例记忆卡片，MVP 可不做。

### 用户输入

首页本身不需要用户输入。用户只需要选择开始记录或进入 Memory Garden。

### AI 输出

首页不需要 AI 实时输出。首页只展示静态产品介绍、边界说明和流程说明。

### 页面跳转逻辑

- 点击“开始今天的记录”跳转到 AI Companion Chat。
- 点击“进入 Memory Garden”跳转到 Memory Garden。
- 点击顶部 Logo 或产品名留在首页。
- 用户点击语音输入按钮 → 检查麦克风权限 → 授权成功后开始录音 → 语音转文字后进入聊天记录。
- 如果用户拒绝麦克风权限 → 停留在当前页面，并提示用户使用文字输入。

## 2. AI Companion Chat

### 页面目标

AI Companion Chat 用于完成情绪倾诉。用户可以从一张图片、一条语音、一句话或一段混乱表达开始，AI 以朋友式陪伴进行回应、追问和阶段性整理。

AI 的目标不是判断用户状态，也不是给治疗建议，而是帮助用户把今天发生的事情和真实感受慢慢说清楚。

### 页面布局

- 顶部区域：页面标题“今天想和我说些什么？”和副标题“可以进行语音输入，可以写下今天发生的事，也可以直接上传一张图片。”。
- 中间区域：聊天记录列表，区分用户消息、AI 消息和图片消息。
- 图片预览区：用户上传图片后展示缩略图，可删除或替换。
- 底部输入区：包含语音输入按钮、文字输入框、图片上传按钮、发送按钮。
- 辅助操作区：包含“换个问题问我”和“我说完了，生成日记”。

### 主要按钮

- 上传图片：上传当天照片、截图、风景、自拍或其他记忆素材。
- 发送：发送用户输入的语音内容、文字。
- 换个问题问我：让 AI 换一种更轻松的问题继续引导。
- 我说完了，生成日记：结束对话并进入 Diary Result / Mood Check-in。
- 返回首页：如果当前对话未保存，需要弹出离开提醒。

### 用户输入

用户可以输入：

- 今天发生的事情。
- 自己的情绪感受。
- 一张图片。
- 一句很短的话。
- 一段混乱的倾诉。
- 对 AI 问题的回答。

输入框占位文案建议：“慢慢说，我在听。今天发生了什么？”

### AI 输出

AI 输出分为三类：

1. 温柔回应：承接用户感受，避免评判。
2. 轻柔追问：帮助用户继续表达某个瞬间、感受或需要。限制追问次数在3-4次，之后不再继续追问。
3. 阶段性整理：把用户已表达内容简短归纳，询问是否准确。

AI 输出规则：

- 不做心理诊断。
- 不提供治疗或医疗建议。
- 不使用绝对化判断。
- 不强迫用户继续回答。

### 页面跳转逻辑

- 点击“我说完了，生成日记”进入 Diary Result / Mood Check-in。
- 如果用户上传了图片，图片作为日记素材并优先用于记忆卡片封面。
- 如果用户没有上传图片，后续由 AI 根据日记生成风格化封面。
- 点击返回且对话未保存时，弹出提示：“这段记录还没有保存，确定离开吗？”。

## 3. Diary Result / Mood Check-in

### 页面目标

Diary Result / Mood Check-in 用于把对话内容转化为完整、温柔、结构化的日记，并让用户选择情绪、情绪底色和封面，最终保存为 Memory Garden 中的一张记忆卡片。

### 页面布局

- 顶部区域：返回按钮、页面标题“今日日记已整理好”。
- 日记内容区：展示标题、日期、今日情绪、正文、情绪总结、记忆摘要。
- 封面图片区：展示用户上传图片或 AI 生成图片。
- 情绪选择区：展示情绪标签和情绪底色选择。
- 操作按钮区：重新生成、调整语气、生成封面、保存到 Memory Garden。

建议日记结构：

- 标题
- 日期
- 今日情绪
- 今天发生的事
- 我真实的感受
- 我想对自己说的话
- 今日的一点点理解

### 主要按钮

- 返回继续聊：回到 AI Companion Chat。
- 重新生成日记：根据原始对话重新生成一版。
- 更温柔一点：让 AI 改写成更柔和的语气。
- 更简短一点：压缩内容。
- 更像真实日记：让文字更自然、更像用户自己写的。
- 选择情绪底色：打开颜色选择面板。
- 生成记忆封面：生成或重新生成封面图。
- 保存到 Memory Garden：保存为记忆卡片并跳转。

### 用户输入

用户可以输入或选择：

- 情绪标签：开心、平静、疲惫、焦虑、难过、委屈、愤怒、释然、空白。
- 情绪底色：低饱和颜色，用于卡片视觉。
- 直接编辑日记正文。
- 给 AI 的改写指令，例如“不要太煽情”“更像我一点”“把重点放在今天的压力上”。

### AI 输出

AI 输出包括：

- 结构化日记。
- 情绪总结。
- 记忆卡片摘要。
- 封面图片提示词或生成结果。

AI 输出需要保持“记录和陪伴”定位，不生成诊断、治疗建议或医疗判断。

### 页面跳转逻辑

- 从 AI Companion Chat 点击“生成日记”进入本页。
- 点击“返回继续聊”回到 AI Companion Chat。
- 点击“保存到 Memory Garden”保存成功后跳转到 Memory Garden。
- 点击“重新生成日记”留在当前页面并刷新日记内容。
- 点击“生成记忆封面”留在当前页面并刷新封面。

## 4. Memory Garden

### 页面目标

Memory Garden 用于展示用户保存过的所有日记记忆。页面不应像普通列表，而应像情绪花园或回忆相册。重点是让用户通过封面图、日期和情绪快速回忆过去。

### 页面布局

- 顶部区域：标题“Memory Garden”和副标题“这里收藏着你认真度过的每一天。”。
- 右上角操作：写下今天、筛选情绪。
- 主体区域：以卡片网格或横向卡片流展示记忆。
- 卡片内容：封面图、日期、标题、主要情绪、情绪底色。
- 筛选区：按情绪筛选、按月份筛选、关键词搜索。MVP 可先保留时间排序和情绪筛选。
- 空状态：当没有记忆卡片时，展示“还没有记忆，写下今天吧”。

### 主要按钮

- 写下今天：进入 AI Companion Chat。
- 查看全部：清除筛选条件，展示全部卡片。
- 按情绪筛选：筛选某种情绪的日记卡片。
- 点击某张记忆卡片：进入 Memory Detail / Past Self Chat。
- Home：返回首页。

### 用户输入

MVP 阶段用户可以输入或选择：

- 搜索关键词。
- 情绪筛选。
- 月份筛选。

搜索框占位文案建议：“搜索某一天、某种心情，或一个关键词……”

### AI 输出

Memory Garden MVP 不需要大量 AI 输出。基础版本只展示保存过的记忆卡片。

后续增强可以加入 AI 总结区域，例如“最近 7 天，你记录最多的情绪是疲惫和平静”，但 MVP 暂不做复杂趋势分析。

### 页面跳转逻辑

- 点击“写下今天”进入 AI Companion Chat。
- 点击某张记忆卡片进入 Memory Detail / Past Self Chat。
- 点击 Home 返回首页。
- 用户筛选情绪后，当前页面更新卡片列表。
- 用户搜索关键词后，当前页面显示匹配卡片。

## 5. Memory Detail / Past Self Chat

### 页面目标

Memory Detail / Past Self Chat 是产品的特色页面。用户可以查看过去某一天的日记，并与“基于这篇日记生成的过去自我画像”对话，完成温柔复盘。

该页面不是让 AI 冒充真实用户本人，而是基于日记内容、情绪、图片和关键词生成一个回忆视角，帮助用户重新理解那一天。

### 页面布局

建议采用左右分栏：

- 左侧为记忆详情区：展示封面图、日期、日记标题、当日情绪、情绪底色、完整日记内容、关键词。
- 右侧为对话区：标题“和那天的自己聊聊”，说明 AI 会根据这篇日记模拟那天的情绪状态陪用户回看。
- 推荐问题区：展示几个可点击问题。
- 底部输入区：用户输入问题或想对过去自己的话。

移动端可改为上下结构：先展示日记详情，再展示对话区。

### 主要按钮

- 返回 Memory Garden：回到记忆花园页。
- 和那天的我聊聊：开启过去自我对话。
- 生成复盘总结：AI 总结这次对话中的回忆和变化。
- 编辑这篇日记：允许用户修改已保存日记。
- 删除这段记忆：删除当前卡片，需二次确认。
- 重新生成封面：重新生成当前记忆卡片封面。
- 写下今天：进入 AI Companion Chat。

### 用户输入

用户可以输入：

- 对过去自己的问题。
- 想对过去自己的安慰。
- 对那一天的新理解。
- 希望 AI 帮忙复盘的问题。

推荐问题：

- 那天的你最需要什么？
- 现在的我可以对那天的你说什么？
- 你后来有好一点吗？
- 如果重新经历那天，你希望我怎么照顾自己？

### AI 输出

AI 输出分为两种模式：

1. 过去的我：基于日记内容，以“那天的我”的语气回应。
2. 复盘陪伴者：帮助用户总结过去和现在的变化。

输出限制：

- 必须基于当前日记内容生成。
- 必须说明这是“过去自我画像”，不是用户本人的真实意识。
- 不输出诊断、治疗或医疗建议。
- 不用“你一定”“你必须”等强制语气。

### 页面跳转逻辑

- 从 Memory Garden 点击记忆卡片进入 Memory Detail。
- 点击“返回 Memory Garden”回到 Memory Garden。
- 点击“写下今天”进入 AI Companion Chat。
- 点击“生成复盘总结”在当前页面生成总结。
- 点击“删除这段记忆”弹出确认框，确认后返回 Memory Garden。
