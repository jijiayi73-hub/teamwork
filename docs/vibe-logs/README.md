## 使用指南

每个日志文件应包含：

1. **日期和分支**：记录创建时的分支状态
2. **用户请求**：本次任务的具体需求
3. **源文档引用**：相关的 Markdown 设计文档
4. **文件变更**：涉及的代码文件列表
5. **关键决策**：技术选型和架构约束
6. **验证结果**：测试命令和执行结果
7. **风险和阻塞**：当前存在的问题
8. **下一步计划**：后续迭代方向
