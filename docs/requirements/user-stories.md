# User Stories

Record user roles, user stories, and acceptance criteria here.

## 1. 用户角色

### 1.1 普通用户 User

普通用户是产品的主要使用者。TA 希望用低压力的方式记录情绪，与 AI 进行温柔对话，把当天的表达整理成日记，并在 Memory Garden 中回看过去的自己。

### 1.2 管理员 Admin

管理员主要服务于课程项目演示和系统维护。TA 关注功能是否可用、数据是否正常生成、接口是否稳定，不负责心理诊断、心理干预或用户情绪评判。

## 2. 普通用户 User 用户故事

### US-01 首页理解产品

作为普通用户，我希望进入首页后能快速理解这是一个 AI 陪伴式情绪日记产品，这样我可以判断它是否适合我记录今天的心情。

验收标准：

- 首页展示产品名称“心灵情绪日记 / Mindful Memory Diary / Inner Garden”。
- 首页展示一句清晰说明：用户可以倾诉、AI 会陪伴整理，并生成记忆卡片。
- 首页展示边界说明：本产品不是心理诊断工具，只做情绪记录、陪伴和回忆整理。
- 首页提供“开始今天的记录”和“进入 Memory Garden”两个主要入口。

### US-02 开始今日记录

作为普通用户，我希望从首页一键开始今天的情绪记录，这样我不需要在复杂菜单里寻找入口。

验收标准：

- 点击“开始今天的记录”后进入 AI Companion Chat。
- 页面标题或提示语能引导用户开始表达，例如“今天想和我说些什么？”。
- 输入框占位文案温柔、低压力，例如“慢慢说，我在听。今天发生了什么？”。

### US-03 文字倾诉

作为普通用户，我希望可以输入今天发生的事情和自己的感受，这样 AI 可以根据我的表达陪我整理情绪。

验收标准：

- 用户可以在对话页输入文字并发送。
- 发送后用户消息出现在聊天记录中。
- AI 返回温柔回应，不输出诊断、治疗或医疗建议。
- AI 可以提出轻柔追问，帮助用户继续表达。

### US-04 上传图片作为记忆素材

作为普通用户，我希望可以上传一张今天相关的图片，这样这张图片可以成为日记和记忆卡片的一部分。

验收标准：

- 对话页提供“上传图片”入口。
- 上传后页面显示图片预览。
- 生成日记时，图片可作为日记素材。
- 保存记忆卡片时，用户上传图片优先作为封面素材。

### US-05 换一个问题

作为普通用户，当 AI 问题让我不知道怎么回答时，我希望可以让 AI 换一种问法，这样对话不会卡住。

验收标准：

- AI 提问后页面提供“换个问题问我”按钮。
- 点击后 AI 生成一个新的、更轻松的问题。
- 新问题仍围绕今日事件、感受或回忆整理，不转向心理诊断。

### US-06 生成结构化日记

作为普通用户，我希望在说完之后让 AI 帮我生成一篇结构化日记，这样我不需要自己重新组织语言。

验收标准：

- 对话页提供“我说完了，生成日记”按钮。
- 点击后进入 Diary Result / Mood Check-in 状态。
- 生成结果包含标题、日期、正文、今日情绪、情绪总结、记忆摘要。
- 日记语气温柔、自然，像帮助用户整理回忆，而不是评价用户。

### US-07 编辑 AI 生成内容

作为普通用户，我希望可以修改 AI 生成的日记，这样最终保存的内容更像我自己的表达。

验收标准：

- 日记正文区域可编辑。
- 用户可以要求 AI“更温柔一点”“更简短一点”“更像真实日记”。
- 修改后页面保留用户最终确认版本。
- 保存时以用户最终确认版本为准。

### US-08 选择情绪和情绪底色

作为普通用户，我希望给今天选择一个情绪标签和颜色，这样以后在 Memory Garden 中更容易回忆这一天。

验收标准：

- Diary Result 页面提供情绪选择，例如开心、平静、疲惫、焦虑、难过、委屈、愤怒、释然、空白。
- 页面提供情绪底色选择。
- 选择结果展示在日记预览和记忆卡片预览中。
- 情绪标签只用于记录和回顾，不生成诊断结论。

### US-09 生成或选择记忆封面

作为普通用户，我希望系统能为日记生成一张治愈风格封面，这样我的记忆卡片更有画面感。

验收标准：

- 如果用户上传过图片，系统默认使用上传图片作为封面素材。
- 如果用户没有上传图片，系统根据日记内容生成风格化封面。
- 封面风格符合治愈系水彩、淡油画、梦境花园插画等视觉方向。
- 用户可以重新生成封面或保留当前封面。

### US-10 保存到 Memory Garden

作为普通用户，我希望把今天的日记保存成记忆卡片，这样以后可以在 Memory Garden 中回看。

验收标准：

- Diary Result 页面提供“保存到 Memory Garden”按钮。
- 保存后系统生成一张记忆卡片。
- 记忆卡片至少包含封面图、日期、标题和主要情绪。
- 保存成功后跳转到 Memory Garden，并能看到新增卡片。

### US-11 浏览 Memory Garden

作为普通用户，我希望像逛花园一样浏览过去的记忆卡片，这样回忆不只是文字列表，而更有情绪氛围。

验收标准：

- Memory Garden 展示所有已保存记忆卡片。
- 卡片按时间排序，默认最新记录优先。
- 每张卡片展示封面、日期、标题、主要情绪。
- 页面提供“写下今天”入口。

### US-12 筛选记忆卡片

作为普通用户，我希望按情绪或时间筛选记忆卡片，这样我可以快速找到某一类回忆。

验收标准：

- MVP 至少支持按时间排序。
- 可提供按情绪筛选入口。
- 筛选后卡片列表只展示符合条件的记忆。
- 清除筛选后恢复全部卡片。

### US-13 查看记忆详情

作为普通用户，我希望点击某张记忆卡片后查看完整日记，这样我可以重新进入那一天的回忆。

验收标准：

- 点击 Memory Garden 中的卡片后进入 Memory Detail。
- 详情页展示封面图、日期、标题、完整日记、情绪、情绪底色和关键词。
- 详情页提供“返回 Memory Garden”按钮。
- 详情页提供“和那天的我聊聊”入口。

### US-14 和过去的我对话

作为普通用户，我希望可以和“基于这篇日记生成的过去自我画像”对话，这样我可以用更温柔的方式回看当时的自己。

验收标准：

- 点击“和那天的我聊聊”后开启对话区。
- AI 回复基于当前日记内容、情绪和关键词生成。
- 页面说明 AI 是基于日记生成的过去自我画像，不冒充真实用户本人。
- AI 回复保持回忆复盘和陪伴语气，不输出心理诊断或治疗建议。

### US-15 生成复盘总结

作为普通用户，我希望对话结束后生成一段复盘总结，这样我可以看到现在的自己与过去的自己的连接。

验收标准：

- Memory Detail 页面提供“生成复盘总结”按钮。
- 总结内容围绕回忆、情绪变化和自我理解。
- 总结不包含诊断、治疗或医疗建议。
- 用户可以选择保存或不保存该总结。

## 3. 管理员 Admin 用户故事

### AS-01 查看基础统计

作为管理员，我希望查看系统的基础统计数据，这样我可以在课程答辩中说明项目是否跑通核心流程。

验收标准：

- 管理员可以查看总对话数、总日记数、总记忆卡片数。
- 管理员可以查看近 7 天新增记录数。
- 统计数据不包含心理诊断字段。
- 统计数据不直接展示用户完整私密日记正文。

### AS-02 查看系统状态

作为管理员，我希望查看前后端和 AI 服务配置状态，这样我可以确认演示时系统是否可用。

验收标准：

- 管理员可以看到后端健康检查状态。
- 管理员可以看到 AI 对话、日记生成、图片生成等服务的可用状态。
- 服务异常时页面给出明确错误提示。
- 错误提示面向系统维护，不对用户情绪做判断。

### AS-03 查看功能使用概览

作为管理员，我希望了解用户主要使用了哪些功能，这样可以评估 MVP 重点是否符合预期。

验收标准：

- 管理员可以查看文字输入次数、图片上传次数、保存卡片次数。
- 管理员可以查看 Memory Detail 打开次数和过去自我对话次数。
- 数据用于课程项目分析，不用于评判用户心理状态。

## 4. 页面维度用户故事汇总

### 4.1 Home

用户故事：

- 作为普通用户，我希望首页安静、清楚、低压力，让我愿意开始记录。
- 作为普通用户，我希望首页能直接进入记录或回忆花园。

验收标准：

- 首页展示产品定位、边界说明和两个主要入口。
- 点击“开始今天的记录”进入 AI Companion Chat。
- 点击“进入 Memory Garden”进入记忆花园。

### 4.2 AI Companion Chat

用户故事：

- 作为普通用户，我希望 AI 像朋友一样听我说今天发生了什么。
- 作为普通用户，我希望可以输入文字或上传图片。

验收标准：

- 支持文字发送和图片上传。
- AI 回复包含温柔回应、轻柔追问或阶段性整理。
- 页面提供生成日记入口。

### 4.3 Diary Result / Mood Check-in

用户故事：

- 作为普通用户，我希望 AI 把我的倾诉整理成日记。
- 作为普通用户，我希望可以选择情绪和封面，再决定是否保存。

验收标准：

- 页面展示结构化日记。
- 支持编辑、重新生成、选择情绪、选择底色、生成封面。
- 保存后进入 Memory Garden。

### 4.4 Memory Garden

用户故事：

- 作为普通用户，我希望通过卡片和封面回看过去的情绪记忆。
- 作为普通用户，我希望可以从这里继续写下今天。

验收标准：

- 页面展示全部记忆卡片。
- 卡片点击后进入详情页。
- 页面提供“写下今天”按钮。

### 4.5 Memory Detail / Past Self Chat

用户故事：

- 作为普通用户，我希望查看某一天的完整日记。
- 作为普通用户，我希望和“那天的我”聊聊，完成温柔复盘。

验收标准：

- 页面展示完整记忆详情。
- 支持开启过去自我对话。
- 支持生成复盘总结。
- 支持返回 Memory Garden。

## 当前第一阶段前端用户故事补充

## 第一阶段用户故事

### 1. 开始一次记录

作为普通用户，我希望打开记录页后可以直接开始写今天发生的事，这样我不需要先理解复杂功能。

验收标准：

- 进入记录页后，前端能创建或持有一个 `conversation_id`。
- 页面显示文本输入框和发送按钮。
- 页面没有当前会话时，会调用 `POST /api/v1/conversations`。

### 2. 发送文字并得到陪伴回复

作为普通用户，我希望每发送一段文字后，都能看到系统给我的陪伴式回复，这样记录过程不是冷冰冰的表单填写。

验收标准：

- 输入非空文字后可以点击发送。
- 发送成功后，页面同时展示用户消息和后端返回的 `assistant_message`。
- 消息按时间顺序展示。
- 发送失败时，页面提示失败并允许用户重试。

### 3. 连续记录多条内容

作为普通用户，我希望可以连续发送多条文字，这样我能慢慢把一天发生的事情说完整。

验收标准：

- 同一页面内可以连续发送多条消息。
- 每次发送都使用同一个 `conversation_id`。
- 页面不会因为重复发送而丢失之前的消息。

### 4. 生成日记

作为普通用户，我希望在说完之后点击“生成日记”，让系统把这次对话整理成一篇日记。

验收标准：

- 至少发送过一条用户消息后，生成日记按钮可用。
- 点击后调用 `POST /api/v1/conversations/{conversation_id}/diary`。
- 生成中有 loading 状态，避免重复点击。
- 成功后展示日记标题、正文、心情和摘要。

### 5. 查看历史日记

作为普通用户，我希望能在历史页看到已经生成的日记，这样我可以回顾以前的记录。

验收标准：

- 历史页调用 `GET /api/v1/diaries`。
- 列表展示标题、摘要、心情和创建时间。
- 没有日记时展示空状态。
- 请求失败时展示错误状态。

### 6. 查看日记详情

作为普通用户，我希望点击历史列表中的日记后能看到完整正文。

验收标准：

- 详情页调用 `GET /api/v1/diaries/{diary_id}`。
- 页面展示标题、正文、心情、摘要、创建时间和更新时间。
- 正文换行应正常显示。
- 页面提供返回历史页的入口。
